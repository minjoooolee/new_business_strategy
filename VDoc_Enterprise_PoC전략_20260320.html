1PoC의 단 하나의 목표

검증 명제
"건설업 대표가 매월 3분 투자하면, 중대재해법 의무이행 증빙이 자동으로 쌓인다"
확인할 것은 딱 세 가지다: 대표가 돈을 내는지, 직원이 참여하는지, 증빙이 법적으로 의미가 있는지. 나머지는 전부 PoC 이후다.
HYPOTHESIS 1

WTP — 대표가 돈을 내는가?

검증: 파일럿 계약 체결 여부
3개월 무상이지만, 계약서에 "4개월차부터 월 45만 원" 명시
무상만 하고 유상 전환 의사 없으면 → 가설 실패
이 가설이 실패하면 사업 자체를 피벗
HYPOTHESIS 2

참여율 — 직원이 하는가?

목표: 스크리닝 참여율 60% 이상
HR 공지 + 리마인더 1회 방식
법정의무교육 수준의 참여율 달성 여부
실패 시 접근 방식 재설계 (사업은 계속)
HYPOTHESIS 3

증빙 — 법적으로 되는가?

검증: 3단계 전자결재 완료 여부
경영자 지시사항 기입 여부
생성된 PDF가 법적 증빙으로 인식되는지
에쓰오일 불기소 사례의 "실질적 이행" 패턴

2핵심 기술 구성 — 만드는 것 vs 안 만드는 것

2.1 PoC에 들어가는 4대 모듈

MODULE 1

직원 스크리닝 웹

형태: 모바일 반응형 웹 (앱 X)
진입: 카카오톡/문자 링크 클릭
구성 (약 20문항, 5~7분):
① 동의 화면 (민감정보 + 익명통계)
② PHQ-9 (9문항) — 우울
③ GAD-7 (7문항) — 불안
④ 신체건강 간이체크 (3문항)
채점: 룰베이스 if-else
PHQ-9: 0~4 정상 / 5~9 경증 / 10~14 중등도 / 15~27 중증
결과: 본인만 확인, "주의 이상" 시 외부 상담 안내
MODULE 2

CEO 월간 레포트

형태: PDF 1장 (스크롤 없이 한 화면)
생성: 반자동 (50% 수동 허용)
포함 항목:
① 전사 종합지수 (정신+신체+참여율)
② 위험도 분포 (양호/관심/주의/위험)
③ 부서별 비교 (10명 미만 통합)
④ 참여율
⑤ 핵심 3줄 요약 + 권고 사항
⑥ 전자결재란
안 넣는 것: 트렌드 그래프, ROI, 이직률 예측
MODULE 3 · 핵심

전자결재 (3단계)

이것이 PoC의 핵심이다
3단계 구조:
① 담당(HR): 레포트 확인 + 서명 + 미이행 사유 입력
② 검토(안전보건담당자): 시행령 제5조 체크리스트 3항목 + 서명
③ 승인(경영책임자): 3줄 요약 확인 + 조치 지시사항 입력 + 서명
완료 시:
PDF 확정 + SHA-256 해시값 + 타임스탬프 + 5년간 삭제/수정 불가
→ 전자문서법 제4조의2 충족
MODULE 4

익명화 · 프라이버시

k-익명성: 10명 기준
10명 미만 부서 → 상위 조직 통합
교차 필터 없음 (부서 1차원만)
개인 데이터:
원본 AES-256 암호화 저장
대표/HR 포함 누구도 열람 불가
대시보드에는 집계 통계만 표시

2.2 PoC에서 의도적으로 빼는 것

빼는 것 — Phase 1 이후 구현
• 직원 앱 부가 기능 전부 (지난 결과, 트렌드, AI 건강 비서, 교육 콘텐츠)
• 실시간 대시보드 (CEO는 월 1회 PDF만 수신)
• 건강검진 데이터 연동 및 AI 해석
• 근태 데이터 연동 (HR 시스템 연동, 이직률 상관분석)
• 고위험자 자동 탐지 → EAP 자동 연계 파이프라인
• 건강친화기업 인증 기능 / ESG 리포트
빼는 이유 — PoC의 원칙
"PoC는 제품을 만드는 것이 아니라, 사업 가설을 검증하는 것이다."
대시보드가 아무리 화려해도 대표가 돈을 안 내면 무의미하다. 교육 콘텐츠가 풍부해도 직원이 스크리닝을 안 하면 무의미하다. 최소 기능으로 핵심 가설 3개만 검증하고, 나머지는 가설이 통과된 후 만든다.
트렌드 분석, 상관관계, ROI 계산 등은 데이터가 3개월 이상 축적된 후에야 의미가 있다. PoC 3개월로는 물리적으로 불가능하다.

3전자결재가 PoC의 핵심인 이유

법적 맥락 — 중대재해처벌법 제4조 · 시행령 제5조
경영책임자의 안전보건확보의무 핵심은 "유해·위험요인 확인·개선절차 마련"과 "안전보건관리 책임자 등 평가기준 마련"이다. 단순히 제도를 갖추는 것이 아니라, "경영책임자가 실질적으로 관여하고 있다"는 증거가 있어야 한다.

4필수 KPI — PoC 성과 측정 지표

#KPI목표값측정 방법연결 가설판단 기준
K1 유상 전환 의사 계약서 서명 (월 45만 원) 파일럿 종료 후 유상 계약 체결 여부 가설 1 (WTP) Yes/No — 이것이 Go/No-Go 결정
K2 스크리닝 참여율 ≥ 60% 응답 완료 인원 / 전체 직원 수 × 100 가설 2 (참여) 60% 미만 → 접근 방식 재설계
K3 전자결재 완료율 100% (매월) 3단계 결재 완료 건수 / 발행 건수 가설 3 (증빙) 미완료 → 결재 프로세스 개선
K4 경영자 지시사항 기입률 100% 지시사항 텍스트 입력 여부 가설 3 (증빙) 미기입 → UX 개선 또는 교육
K5 스크리닝 완료 시간 ≤ 7분 시작~완료 소요 시간 중앙값 가설 2 (참여) 7분 초과 → 문항 간소화
K6 증빙 만족도 ≥ 4.0/5.0 파일럿 종료 후 고객 설문 가설 3 (증빙) "이 PDF가 법적으로 도움이 되는가?"
KPI 우선순위
K1(유상 전환 의사)이 유일한 Go/No-Go 지표다. K2~K6는 제품 개선 방향을 알려주는 보조 지표이지만, K1이 실패하면 나머지가 아무리 좋아도 의미가 없다. 대표가 돈을 안 내면 사업이 없다.

5사업성 평가 — PoC가 답하는 질문들

사업성 질문PoC가 제공하는 답판단 기준판단 불가 항목 (Phase 1 이후)
시장이 존재하는가? 실제 계약 체결로 검증
무상→유상 전환 의사 = 최강 증거		 1사 이상 유상 전환 의사 → 시장 존재 시장 규모 추정은 별도 조사 필요
고객이 가치를 느끼는가? 증빙 만족도 + 재사용 의사
"이 PDF가 있으면 밤에 잘 수 있겠다"		 만족도 4.0+ → 가치 인식 확인 NPS, 구전 효과는 6개월 이후
제품이 작동하는가? 참여율 60%+ / 결재 완료 100%
End-to-End 플로우 실제 작동 확인		 2개월 연속 정상 작동 → 기술 검증 확장성, 동시접속, 보안 인증은 이후
가격이 적절한가? 월 45만 원(150인 기준)에 계약하는지 유상 전환 시 가격 저항 여부 관찰 최적 가격점은 10사+ 데이터 필요
반복 매출이 가능한가? 매월 결재를 완료하는지 (습관화) 3개월 연속 결재 → 습관화 가능성 Churn rate는 12개월 이후 측정
확장 가능한가? PoC에서는 판단 불가 — 다업종, 대기업, 해외 등은 Phase 2+
PoC로 판단할 수 있는 것과 없는 것의 경계
PoC 3개월로 "이 사업을 해야 하는가?"는 답할 수 있다. "이 사업이 얼마나 커지는가?"는 답할 수 없다. 전자가 확인되면 후자를 위한 투자가 정당화된다. PoC는 사업의 "시작 여부"를 결정하는 도구이지, "규모"를 예측하는 도구가 아니다.

7Go / Pivot / Stop — 10월의 판정

✅ GO → Phase 1 진입

조건: 3개 가설 중 2개 이상 충족
특히 가설 1(WTP) 충족 필수
다음 단계:
• Phase 1 정식 제품 개발 착수
• 실시간 대시보드 구현
• 건강검진 데이터 연동
• 2호~5호 기업 확보 (제조업 포함)
• 시리즈 투자 또는 사내 추가 투자 논의

⚠️ PIVOT → 방향 조정

조건: 가설 1 충족 + 가설 2 실패
(대표는 돈을 내겠다, 직원이 안 한다)

• 스크리닝 방식 재설계
— 문항 수 축소 (10문항 이내)
— 인센티브 도입 (스크리닝 참여 시 혜택)
— 법정의무교육과 연계
• 사업 자체는 계속 (WTP 확인됨)

🛑 STOP → 사업 재검토

조건: 가설 1 실패
(대표가 돈을 안 낸다)
검토 옵션:
• 세그먼트 변경 (건설→제조, IT)
• 가격 변경 (월 45만→월 25만)
• 가치 제안 변경 (면책→복지/ESG)
• V.Doc Enterprise 자체 피벗
이 판정이 나와도 실패가 아니라 "학습"이다
PoC의 가장 중요한 가치
PoC는 성공하든 실패하든 가치가 있다. 성공하면 사업을 진행할 근거를 얻고, 실패하면 잘못된 방향에 더 큰 자원을 투입하는 것을 막는다. 개발자 1명 + 디자이너 1명 + 8주의 투자로, "이 사업이 될까?"라는 질문에 데이터 기반 답을 얻는 것이 PoC의 본질이다. 기획서 100장보다 고객 1명의 계약서가 더 강력한 증거다.
